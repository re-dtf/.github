# Руководство: Развертывание собственного reDTF

Одно из главных преимуществ архитектуры **reDTF** (100% Client-Side) — возможность абсолютно бесплатно и за пару кликов поднять собственную независимую копию клиента.

Развернув клиент у себя, вы получаете полный контроль над кодом, который исполняется в вашем браузере, и абсолютную уверенность в прозрачности его работы. Ни одна новая строка кода не попадет к вас без спроса.

<hr>

## Шаг 1. Создание форка (вашей копии)

Для начала необходимо скопировать исходный код проекта в ваш личный профиль GitHub.

1. Перейдите в основной репозиторий клиента: [**re-dtf/client**](https://github.com/re-dtf/client).
2. В правом верхнем углу интерфейса GitHub нажмите кнопку **Fork**.
3. Оставьте настройки по умолчанию и нажмите **Create fork**.

<blockquote>
  <b>Примечание о маршрутизации:</b> По умолчанию GitHub Pages размещает сайты в подпапках (например, <code>https://ваше-имя.github.io/имя-репозитория/</code>). Если вы хотите, чтобы клиент открывался прямо в корне домена (<code>https://ваше-имя.github.io</code>), переименуйте репозиторий форка в точности как <code>ваше-имя.github.io</code> (где "ваше-имя" — логин вашего аккаунта).
</blockquote>

<hr>

## Шаг 2. Активация GitHub Actions

По умолчанию GitHub отключает автоматическое выполнение скриптов сборки (Actions) в форках из соображений безопасности. Их необходимо включить вручную.

1. Перейдите в вашем свежесозданном форке на вкладку **Actions** (в верхнем навигационном меню).
2. Вы увидите предупреждение безопасности.
3. Нажмите зеленую кнопку <b>«I understand my workflows, go ahead and enable them»</b>.

<hr>

## Шаг 3. Настройка источника GitHub Pages

Теперь нужно указать платформе, откуда брать скомпилированные файлы для хостинга.

1. Перейдите на вкладку **Settings** в вашем форке.
2. В левом боковом меню выберите раздел **Pages**.
3. В блоке <i>Build and deployment</i> найдите выпадающий список <b>Source</b>.
4. Измените его значение с <code>Deploy from a branch</code> на <b><code>GitHub Actions</code></b>.

<hr>

## Шаг 4. Запуск первой сборки (Деплой)

Система настроена, осталось инициировать процесс компиляции.

1. Снова перейдите на вкладку **Actions**.
2. В левом боковом меню, в разделе <i>All workflows</i>, выберите воркфлоу сборки (обычно называется <code>Deploy to GitHub Pages</code>).
3. В правой части экрана появится серая панель. Нажмите <b>Run workflow</b> и подтвердите запуск.
4. Дождитесь завершения процесса (около 1-2 минут). Желтый индикатор сменится зеленой галочкой.

<b>Готово.</b> Ваш личный, независимый клиент reDTF успешно развернут.
Точную ссылку на ваш сайт всегда можно найти на вкладке <b>Settings -> Pages</b> в верхней части экрана.

<hr>

## Поддержание актуальности

Оригинальный клиент reDTF будет получать обновления. Чтобы перенести новые функции и исправления в свою копию, используйте встроенный механизм синхронизации:

1. Зайдите на главную страницу своего форка.
2. Под зеленой кнопкой <code>&lt;&gt; Code</code> найдите статусную строку <i>This branch is X commits behind re-dtf:main</i>.
3. Нажмите <b>Sync fork</b>, затем <b>Update branch</b>.

Сразу после этого GitHub автоматически подтянет свежий код из главного репозитория, а GitHub Actions соберут и опубликуют для вас обновленную версию сайта. Дополнительных действий не требуется.
